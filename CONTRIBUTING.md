# Contributing

Contributions to this project are released to the public under the repository license terms.

Everyone is welcome to contribute by opening issues, improving documentation, suggesting enhancements, or submitting pull requests.

## How to Contribute

### Reporting Issues

If you found a bug or have a feature idea:

1. Check if a similar issue already exists.
2. Open a new issue with:
   - Clear title and short summary
   - Steps to reproduce
   - Expected behavior vs actual behavior
   - Environment details

### Submitting Pull Requests

1. Fork the repository.
2. Create a branch:
   ```bash
   git checkout -b feature/short-description
   ```
3. Run the app and verify changes locally.
4. Keep commits focused and clearly titled.
5. Open a pull request with:
   - Purpose of the change
   - Screenshots (for UI changes)
   - Manual test steps

## Code Style Guidelines

This project uses:

- ASP.NET Web-Forms
- C#
- SQL / LINQ-TO-SQL
- jQuery

Before submitting make sure the project builds warning-free from Visual Studio.

## Coding Standards

1. Keep code focused and readable.
2. Use clear naming for variables and handlers.
3. Separate logic using the Data Access Layer (DAL) appropriately.
4. Keep frontend logic minimal with jQuery.
5. Document non-obvious logic with short comments.
